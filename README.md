# Dotfiles

Personal development environment configuration for macOS and Linux, managed with
[GNU Stow](https://www.gnu.org/software/stow/). Uses **zsh** as the default
shell on both platforms.

## Quick Start

```bash
# 1. Clone with submodules
git clone --recurse-submodules git@github.com:catpaladin/dotfiles.git
cd dotfiles

# 2. Run setup (stows configs, configures .zshrc)
./setup.sh

# 3. Symlink Neovim config manually (not stowed — see Neovim section)
mkdir -p ~/.config
ln -s "$(pwd)/nvim/.config/nvim" ~/.config/nvim

# 4. Install dev environments (optional, pick what you need)
./scripts/install-core-tools.sh   # core CLI tools the aliases depend on
./scripts/python-dev.sh
./scripts/go-dev.sh
./scripts/rust-dev.sh
./scripts/terraform-dev.sh
./scripts/typescript-dev.sh

# 5. Restart your shell
exec zsh
```

## Prerequisites

| Tool | Purpose | Install |
|------|---------|---------|
| [GNU Stow](https://www.gnu.org/software/stow/) | Symlink farm management | `brew install stow` / `apt install stow` |
| [Nerd Fonts](https://www.nerdfonts.com/) v3+ | Icons in terminal & tmux | `brew install --cask font-hack-nerd-font` (macOS) or `./scripts/install-font-linux.sh` (Linux) |
| [eza](https://github.com/eza-community/eza) | Modern `ls` replacement (`ls`/`ll`/`la`/`tree` aliases) | `brew install eza` / `apt install eza` (Ubuntu 24.04+; otherwise see [eza docs](https://github.com/eza-community/eza#installation)) |
| [ripgrep](https://github.com/BurntSushi/ripgrep) | Recursive search (`grep` alias → `rg`) | `brew install ripgrep` / `apt install ripgrep` (also installed by `./scripts/rust-dev.sh`) |
| [fzf](https://github.com/junegunn/fzf) | Fuzzy finder (Ctrl-R, Ctrl-T, Alt-C) | `brew install fzf` / `apt install fzf` |
| [fd](https://github.com/sharkdp/fd) | Fast file finder (used by fzf) | `brew install fd` / `apt install fd-find` |
| [bat](https://github.com/sharkdp/bat) | Syntax-highlighted file viewer (suffix aliases) | `brew install bat` / `apt install bat` |
| [starship](https://starship.rs/) | Cross-shell prompt | `brew install starship` / `curl -sS https://starship.rs/install.sh \| sh` |
| [jless](https://github.com/PaulJuliusMartinez/jless) | Interactive JSON viewer (`.json` suffix alias) | `brew install jless` |

> [!NOTE]
> The setup script will warn and skip gracefully if some tools are missing,
> but the shell integrations (fzf, starship, suffix aliases) won't activate
> until the tools are installed. Run `./scripts/install-core-tools.sh` to
> bootstrap all of the core CLI tools the zsh aliases depend on (eza, ripgrep,
> bat, fd, fzf, starship, jless).

## Zsh Modules

The `.zshrc` file is minimal — it just sources every `*.zsh` file in
`~/.zsh.d/` alphabetically. Each module is self-contained and gracefully
degrades if its tool isn't installed.

### Module Overview

| Module | What it does |
|--------|-------------|
| `common_aliases.zsh` | `eza`-based `ls`/`ll`/`la`/`tree`, `rg`-backed `grep`, `kubectl` alias, suffix aliases for file-type-based opening |
| `core.zsh` | History settings, oh-my-zsh plugin loading (autosuggestions, syntax-highlighting) |
| `editor.zsh` | Sets `$EDITOR=nvim`, binds Ctrl-X Ctrl-E to edit-and-execute |
| `fzf.zsh` | fzf shell integration, fd integration, Tokyo Night theme, Tab completion |
| `functions.zsh` | `dedupe_path()` and `cover()` (Go test coverage) helper functions |
| `paths.zsh` | Conditional PATH setup for all toolchains (cargo, goenv, tfenv, nvm, bun, brew) |
| `prompt.zsh` | Starship prompt initialization |

### Shell Shortcuts & Aliases

#### Key Bindings

| Shortcut | Action |
|----------|--------|
| `Ctrl-R` | Fuzzy search command history (fzf) |
| `Ctrl-T` | Fuzzy file picker — pastes selected paths into command line |
| `Alt-C` | Fuzzy directory cd |
| `Ctrl-X Ctrl-E` | Edit current command line in Neovim, execute on `:wq` |
| `Ctrl-/` | Toggle preview window in fzf |

#### Suffix Aliases

Type a filename without a command prefix and zsh opens it with the right tool:

| Extension(s) | Opens with | Example |
|---------------|-----------|---------|
| `.json` | `jless` (interactive pager) | `package.json` → `jless package.json` |
| `.yaml` `.yml` `.toml` `.ini` `.conf` `.cfg` `.env` `.properties` | `bat` | `docker-compose.yml` → `bat docker-compose.yml` |
| `.py` `.go` `.rs` `.ts` `.tsx` `.js` `.jsx` `.lua` | `bat` | `main.go` → `bat main.go` |
| `.sh` `.bash` `.zsh` | `bat` | `setup.sh` → `bat setup.sh` |
| `.tf` `.tfvars` `.hcl` | `bat` | `main.tf` → `bat main.tf` |
| `.css` `.html` `.xml` `.md` `.txt` `.log` `.csv` `.tsv` | `bat` | `README.md` → `bat README.md` |

To override a suffix alias for a single use, prefix with a command:
`cat file.yml`, `vim main.go`, etc.

#### Standard Aliases

| Alias | Command | Requires |
|-------|---------|----------|
| `ls` | `eza --icons` | eza |
| `ll` | `eza -lh --icons --git` | eza (+ git for the `--git` column) |
| `la` | `eza -lah --icons --git` | eza (+ git) |
| `tree` | `eza --tree --icons` | eza |
| `grep` | `rg --color=auto` | ripgrep |
| `diff` | `diff --color=auto` | coreutils |
| `k` | `kubectl` | kubectl |

> [!NOTE]
> The `ll`/`la` aliases pass `--git` to eza, which adds a per-file git
> status column. Outside a git repo (or without git installed) the column is
> simply empty — the aliases still work.

### PATH Management

`paths.zsh` is the single source of truth for PATH. Each toolchain is
guarded with a directory existence check so missing tools don't cause errors:

- `~/.local/bin` — uv tools, tflint, standalone binaries
- `~/.cargo/bin` — Rust / Cargo
- `~/.goenv` — Go via goenv
- `~/.tfenv/bin` — Terraform via tfenv
- `~/.bun/bin` — Bun runtime
- `~/.nvm` — Node.js via nvm
- `/opt/homebrew/bin` — Homebrew (macOS)
- `/home/linuxbrew/.linuxbrew/bin` — Homebrew (Linux)

`dedupe_path` runs last to keep PATH clean.

## Development Environment Scripts

Each script sources `scripts/lib/common.sh` for shared helpers (`print_status`,
`detect_os`, `command_exists`, `add_to_profile`). They are idempotent — safe to
re-run.

### Core CLI Tools

```bash
./scripts/install-core-tools.sh
```

Installs the cross-cutting CLI tools that the zsh aliases and modules depend on
but that aren't covered by the per-language dev scripts. Idempotent and safe to
re-run.

- **eza** — `ls`/`ll`/`la`/`tree` aliases (brew, or apt on Ubuntu 24.04+)
- **ripgrep** (`rg`) — `grep` alias (brew / apt; also installed by `rust-dev.sh`)
- **bat** — suffix aliases for source/config files (brew / apt)
- **fd** — file finder used by fzf (brew / apt; apt ships it as `fdfind`, symlinked to `~/.local/bin/fd`)
- **fzf** — fuzzy finder, Ctrl-R/Ctrl-T/Alt-C (brew / apt)
- **starship** — cross-shell prompt (`curl -sS https://starship.rs/install.sh`)
- **jless** — `.json` suffix alias (brew / apt)

### Python

```bash
./scripts/python-dev.sh
```

Uses **uv** for Python version and package management (replaces pyenv).

- Installs [uv](https://github.com/astral-sh/uv) if not present
- Detects the latest stable Python (including system installs like Homebrew)
- Installs it via `uv python install` if not already available
- Installs CLI tools as isolated uv tools to `~/.local/bin`:
  `ruff`, `black`, `isort`, `mypy`, `pytest`, `ipython`

### Go

```bash
./scripts/go-dev.sh
```

Uses **goenv** for Go version management.

- Installs goenv, initializes it in-session, installs latest stable Go
- Installs: `golangci-lint`, `golines`, `gofumpt`, `goimports-reviser`,
  `gotests`, `godoc`, `cobra-cli`, `air` (live reload), `swag` (API docs)

### Rust

```bash
./scripts/rust-dev.sh
```

Uses **rustup** for Rust toolchain management.

- Installs Rust via rustup
- Installs Python-side Rust tools: `uv`, `ruff`, `rye`
- Installs: `ripgrep`

### Terraform

```bash
./scripts/terraform-dev.sh
```

Uses **tfenv** for Terraform version management.

- Installs tfenv, latest Terraform
- Installs `tflint` (via GitHub release binary to `~/.local/bin` — cross-platform)
- Installs `terraform-docs`

### TypeScript / Node.js

```bash
./scripts/typescript-dev.sh
```

Uses **nvm** for Node.js version management.

- Installs nvm, Node.js LTS, Yarn, Bun
- Installs: `typescript`, `ts-node`, `eslint`, `@typescript-eslint/*`,
  `prettier`, `typedoc`, `tsx`

### Nerd Fonts (Linux only)

```bash
./scripts/install-font-linux.sh
```

Installs Hack Nerd Font and Noto Nerd Font to `~/.local/share/fonts`.

## tmux

The config uses the **Tokyo Night** color palette with a powerline-style
status bar.

| Setting | Value |
|---------|-------|
| Prefix | `Ctrl-F` (remapped from `Ctrl-B`) |
| Shell | `/bin/zsh` |
| History | 10,000 lines |
| Pane sync | `Ctrl-Y` (toggles synchronize-panes) |
| Plugins | tmux-sensible, tmux-resurrect |

### Setup

1. Install [TPM](https://github.com/tmux-plugins/tpm) (Tmux Plugin Manager):
   ```bash
   git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
   ```
2. Start tmux: `tmux`
3. Install plugins: press `Ctrl-F` then `I` (capital I)
4. Reload config: `Ctrl-F` then `r` (if tmux-resurrect is installed) or
   `tmux source-file ~/.tmux.conf`

### Pane Borders

When a window has 2+ panes, a header bar shows the pane index, current
command, and path. Single-pane windows get full height (no header).

## Neovim

Neovim config is tracked as a **git submodule** but is **not stowed** — you
symlink it manually. This allows flexibility between the submodule and a
standalone clone.

### Option A: Use the submodule (recommended)

```bash
# After cloning the dotfiles repo with --recurse-submodules
mkdir -p ~/.config
ln -s "$(pwd)/nvim/.config/nvim" ~/.config/nvim
```

### Option B: Use a standalone clone

If you maintain a separate working clone of your nvim config:

```bash
git clone git@github.com:catpaladin/nvim-config.git ~/projects/nvim-config
ln -s ~/projects/nvim-config ~/.config/nvim
```

Both point to the same upstream repository — the submodule is for version
tracking, the standalone clone is for active editing.

### Updating the submodule

```bash
git submodule update --recursive --remote
```

## Alacritty

Platform-specific configs are stowed automatically based on your OS:

- **macOS**: `alacritty-darwin/` → `~/.alacritty.toml`
- **Linux**: `alacritty-linux/` → `~/.alacritty.toml`

The setup script detects your OS and stows the appropriate package.

## Adding New Configuration

### New zsh module

1. Create a new file in `zsh/.zsh.d/`, e.g. `zsh/.zsh.d/myconfig.zsh`
2. Add it to `.gitignore` with a `!` exception:
   ```
   !zsh/.zsh.d/myconfig.zsh
   ```
3. Re-stow: `stow -R -t ~ zsh`
4. It will be sourced automatically on next shell start (alphabetical order)

### New script

1. Create `scripts/my-dev.sh`
2. Source common helpers at the top:
   ```bash
   SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   source "$SCRIPT_DIR/lib/common.sh"
   detect_os
   ```
3. Use `print_status` / `print_header` for output
4. Use `command_exists` to check for tools
5. Use `add_to_profile` for PATH additions

### New stow package

1. Create a directory at the repo root (e.g. `htop/`)
2. Mirror the target path inside it (e.g. `htop/.config/htop/htoprc`)
3. Add the directory name to the `base` array in `setup.sh`
4. Run `./setup.sh`
